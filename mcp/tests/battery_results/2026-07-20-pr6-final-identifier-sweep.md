## Knee

Decision rule (brief): among **dedup=on** combos, smallest `edge_budget`/`top_k` whose retention is within 5% of the plateau AND whose worst-case size ≤ 9000 est-tokens (≥25% headroom).

**Finding 1 — ceiling saturated; the headroom clause is inapplicable.** Only **0/15** dedup=on combos land ≤ 9000 est-tokens; every combo's worst-case query sits at ~12011 (≈ the 12000 ceiling). Real RAG chunks (~2k tokens each) plus rich edges (~260 tokens each) saturate the ceiling, and the trim fills to just under it regardless of params — so ≥25% headroom is unmeetable at **any** grid point and cannot discriminate. Headroom is a _ceiling_ / chunk-size lever (`EMBEDDINGTON_MAX_RESPONSE_TOKENS`, source_quote/text length), not an `edge_budget`/`top_k` one. The knee is therefore taken on retention + actual edge delivery, holding `top_k` at the shipped 5 (a RAG-breadth lever the KG-only retention metric must not tune).

**Chosen defaults: `edge_budget=120`, `top_k=5`** (dedup=on). Retention 0.425 (plateau 0.425 at top_k=5, ≥ 0.404 threshold), predicate recall 0.500, 11.5 edges delivered — the smallest `edge_budget` at the retention plateau, where delivered edges and predicate diversity have also plateaued; higher budgets add latency without adding edges.

**Differs from the shipped (40, 5)** → this sweep suggests `edge_budget=120` (top_k stays 5); defaults unchanged by this run — apply via a config/docs commit (server.py, enrich.py, RESPONSE_SHAPES.md, CHANGELOG.md) if accepted.

For maximal KG grounding a caller can pass `top_k=3`, which lifts retention to the full-grid peak **0.450** at `edge_budget=120` (fewer vector chunks cede more of the shared ceiling to KG edges). Documented as caller guidance, not a default change — top_k governs the vector half, which the KG-only retention metric does not score.
